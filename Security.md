# Security Structure of the app
the app will be installed on top of dynamics crm. So, it should work with the current powerplatform secrity role structure. 
## User roles
there should be 1 marketing maker, 1 journey designer, 1 app pusher role. The rest will use the OOB Dynamics entities. 
## How users are managed 
the users will be managed with the standard powerplatform user management. Users created int eh AD and then synced to the app and then given a security role.
